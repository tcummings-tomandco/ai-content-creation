# Cloud routine prompt (Option B — write + commit, Action publishes)

This is the prompt for the **remote/cloud** Claude routine (claude.ai Routines, repo `tcummings-tomandco/ai-content-creation`, Tue + Thu 08:00, Opus 4.8).

The cloud routine only WRITES the article and commits it. The Storyblok push is done by the `publish-to-storyblok` GitHub Action, which reads the Storyblok credentials from GitHub Actions secrets. The cloud routine never needs the Storyblok token.

Paste everything between the lines into the routine's Instructions box.

---

Run the Tom & Co AI Content Engine to write the next article and commit it. The repository is already checked out as your working directory. Do not push to Storyblok — a GitHub Action handles publishing.

Read `routines/run.md` for context, then `.claude/skills/tomandco-article/SKILL.md` and its supporting files: `voice-guide.md`, `checklist.md`, `stat-sourcing.md`, `format-mapping.md`. Follow them exactly. Do not skip the QA gate.

Step 1. Pick the topic: `python3 scripts/pick_next_topic.py`. Capture the topic ID and the slug. The picker returns the next open topic in priority order (P0 first, then P1 quick-wins, then P1, P2, P3) and skips any row whose Status is already set, so it never repeats an article.

Step 2. Write the article. Execute SKILL.md sections 1–7:
- Research against UK primary sources only (gov.uk, ICO, FCA, MHRA, CMA, SRA, Ofcom, ONS, Bank of England, BCC, peer-reviewed, named surveys). 5–10 citable datapoints, each with figure + unit + date + source URL. No agency blogs.
- Draft in the friend-who-knows-AI register (see voice-guide.md). British spellings. No em dashes. No banned AI-tell words. Body paragraphs 30–60 words, 70 max. Promote 3+ named sub-topics in a section to H3 sub-headers. Front-load a 40–80 word definitive answer paragraph. Do NOT put a "Last reviewed" line in the body (the date is carried by the article.json `date` field).
- Set the article.json `date` field to today.
- Generate the JSON-LD block.
- Write all outputs to `output/{YYYY-MM-DD}-{slug}/`: article.json, preview.html, sources.md, qa_report.md, internal_link_suggestions.md, stat_bank_update.json (only if a Layer 2 calculation was used), and key_stats.json (MANDATORY — 3–5 punchy stat tiles, per SKILL.md section 7a).

Step 3. QA gate: `python3 scripts/voice_check.py output/{folder}/article.json`. If any check FAILs, iterate the draft and re-run until it reports CLEAN. CLEAN is required. After two failed iteration attempts on the same issue, stop and report it.

Step 4. Rebuild the preview library: `python3 scripts/publish_preview.py {folder}` (folder name only).

Step 5. If a Layer 2 calculation was produced, merge it into the stat bank: append the object in `output/{folder}/stat_bank_update.json` to the `stats` array in `data/stat_bank.json`.

Step 6. Mark the topic as pending publish (this removes it from the queue and signals the Action which article to publish):
```
python3 scripts/mark_topic_done.py {ID} --slug {slug} --status "Draft pending Storyblok"
```

Step 7. Commit and push to main:
```
git config user.email "routine@tomandco.co.uk"
git config user.name "Tom & Co AI Routine"
git add output/{folder}/ data/roadmap.json data/stat_bank.json docs/
git commit -m "article: {slug} — auto-generated by routine, pending Storyblok publish"
git push origin HEAD:main
```

The push to main triggers the `publish-to-storyblok` GitHub Action, which pushes the article to Storyblok as a draft and flips the roadmap row to "Approved — draft in Storyblok". You do not call Storyblok yourself.

Finish with a one-screen summary: topic ID and title, cluster/priority, word count, citation count, stat layer used, QA result, the output folder, and the commit sha. End with: "Committed and pushed. The publish-to-storyblok Action will draft it in Storyblok within a minute or two."

Never fabricate statistics. If element 5 (original UK stat) cannot be filled honestly, flag it in the summary rather than inventing a number. Do not push to the company website repo.

## How the two halves connect

```
Cloud routine (Tue/Thu 08:00, Opus 4.8)
  pick topic -> write article -> QA -> commit output/ + roadmap (Status="Draft pending Storyblok") -> push to main
        |
        v  (push to main triggers)
GitHub Action: publish-to-storyblok
  reads pending rows -> push_to_storyblok (draft) -> flip roadmap row to "Approved — draft in Storyblok" -> commit back
        |
        v
Draft story in Storyblok dashboard -> Tom reviews and publishes
```

Credentials: the Storyblok token lives only in the engine repo's GitHub Actions secrets. The cloud routine has no Storyblok credentials and does not need them.
